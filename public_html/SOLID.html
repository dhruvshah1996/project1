SOLID Principles

S (Single Responsibility)

What is Single Responsibility ?

a function does one thing and one thing only sometimes one thing is to call other function that’s its responsibility

  • To look at an example for this look to the right
  • As you can see the function of calculator class has only one responsibility
  • Its only job is to call the addition, subtraction, multiplication, and division methods
  • It cannot do anything else if it does it mess's up the other classes that are calling this class

O (Open and Close)

What is Open and Close ?

functions and classes should be open for extensions and closed for modifications. For example, it means that if you want to add to it, you can, but you can’t modify it as in you can’t change what the function does, but you can add to it so in the addition function in branch 4 if you want to add a bunch of numbers you can, but you can’t make it, so it adds and subtracts.

  • One of the examples for Open and Close is on the right
  • You can see that we are using the addition method to add a bunch of numbers

L (Liskov Substitution Principle)

What is Liskov Substitution Principle ?

it means that the children class can depend on the parent, but the parent class shouldn’t depend on the child class

  • One of the examples for this is on the right
  • You can see that the multiplication, addition, etc classes have the calculation class imported
  • The reason why is because the calculation class has the method that allows the child classes to do more things
  • Which we see in open and close where the addition method is able to add multiple numbers

I (Interface Segregation)

What is Interface Segregation ?

in OOP for the most part you have what’s called single inheritance which is basically parent child inheritance you can’t have parent child and grandchildren

  • For this we dont really have a good example in our calculator program
  • The reason why is because the idea is to have a class and an interface
  • The interface is supposed to tell the class what it is supposed to do
  • But we dont use interface like that in python much
  • If you look to the right though you will see an example of this
  • Since we dont use interface in python this is a good example since each method is only doing one thing and it is inheriting its properties from the parent class

D (Dependency Inversion Principle)

What is Dependency Inversion Principle ?

rule to prevent your code from being to tied up and be able to have your code to be able to be re used. One of the main thing you should focus when writing code is that make sure your code can be re used

  • The entire calculator program we are following this rule
  • The reason why is because each of our functions and classes are being re used
  • The one class we can take for example is the calculation class
  • Each of the child class is using that parent class to get the properties for them
